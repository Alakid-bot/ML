# ML-based Network Service Load Prediction

This project provides a bilingual machine-learning dashboard for predicting the maximum supported load of network services. It combines a practical training pipeline, a Streamlit web interface, training-history persistence, and Zeabur deployment support.

本项目是一个面向网络服务最大可支持负载预测的双语机器学习系统。它包含训练 pipeline、Streamlit 前端界面、训练历史持久化，以及 Zeabur 部署配置。

## Project Goal / 项目目标

The prediction task is supervised regression:

```text
Input:  resource allocation + traffic + KPI metrics
Output: max_supported_load_mbps
```

The model learns how CPU, RAM, link capacity, latency, throughput, packet loss, and traffic relate to the maximum load a network service can safely support.

本项目将负载预测建模为监督学习回归问题：

```text
输入：资源配置 + 流量 + KPI 指标
输出：max_supported_load_mbps，即最大可支持负载
```

模型学习 CPU、内存、链路容量、延迟、吞吐量、丢包率和输入流量之间的关系，从而预测网络服务在当前条件下能稳定承载的最大负载。

## Training Method / 训练方法

The system trains several candidate regressors and selects the model with the lowest validation RMSE.

系统会训练多个候选回归模型，并根据验证集 RMSE 自动选择最佳模型。

Current candidate models:

当前候选模型包括：

```text
dummy_mean        Mean-value baseline / 均值基线模型
ridge             Ridge regression / 岭回归模型
mlp               MLP neural-network regressor / MLP 神经网络回归模型
adaptive_hybrid   Adaptive hybrid load predictor / 自适应混合负载预测模型
```

### Baseline and Standard Models / 基线与标准模型

`dummy_mean` predicts the mean target value. It is not a useful production model, but it provides a baseline. If a real model cannot beat this baseline, the dataset or feature design is likely weak.

`dummy_mean` 只预测训练集中目标值的平均值。它不是实用模型，而是最低基线。如果真正的模型连它都无法超过，说明数据质量或特征设计可能存在问题。

`ridge` is a stable linear model with L2 regularization. It trains quickly, works well with smaller datasets, and captures the main linear relationship between resources, KPI metrics, and supported load.

`ridge` 是带 L2 正则化的稳定线性模型。它训练快、小数据集表现稳，适合学习资源配置、KPI 指标与最大负载之间的主要线性关系。

`mlp` is a neural-network regressor. It can learn nonlinear patterns, but it needs more data and can overfit when the dataset is small.

`mlp` 是神经网络回归模型，可以学习非线性关系，但它需要更多数据；数据较小时容易过拟合。

### Adaptive Hybrid Model / 自适应混合模型

The dedicated optimized model is `adaptive_hybrid`.

本项目的专属优化模型是 `adaptive_hybrid`。

It combines the advantages of Ridge and MLP:

它结合了 Ridge 和 MLP 的优点：

```text
Ridge backbone: learns the stable main trend
MLP residual correction: learns nonlinear error left by Ridge
Automatic fallback: disables residual correction when data is too small or validation does not improve
```

```text
Ridge 主干：学习稳定的主要趋势
MLP 残差修正：学习 Ridge 没有捕捉到的非线性误差
自动回退：当数据太少或验证集没有提升时，自动关闭残差修正
```

The prediction formula is conceptually:

核心思想可以理解为：

```text
final_prediction = ridge_prediction + residual_weight * mlp_residual_prediction
```

In practice, the model first validates whether the residual correction improves RMSE. If it does not, the final model falls back to Ridge. This gives the model better compatibility across small demo datasets and larger real profiling datasets.

实际训练时，模型会先验证残差修正是否能降低 RMSE。如果没有提升，就自动回退到 Ridge。这样既能兼容小型演示数据，也能在真实数据较多时利用神经网络学习非线性规律。

## Model Artifact / 模型文件如何使用

Training outputs:

训练输出：

```text
load_predictor.joblib
metrics.json
```

`load_predictor.joblib` is a saved scikit-learn Pipeline. It includes preprocessing and the selected model, so prediction code does not need to scale features manually.

`load_predictor.joblib` 是保存好的 scikit-learn Pipeline，包含特征标准化和最终选中的模型。因此使用模型预测时不需要手动做标准化。

Example:

示例：

```python
import joblib
import pandas as pd

model = joblib.load("load_predictor.joblib")

features = pd.DataFrame([
    {
        "traffic_input_mbps": 700,
        "cpu_cores": 4,
        "ram_gb": 8,
        "link_capacity_mbps": 1000,
        "cpu_utilization_percent": 65,
        "memory_utilization_percent": 70,
        "latency_ms": 25,
        "throughput_mbps": 680,
        "packet_loss_percent": 0.3,
    }
])

prediction = model.predict(features)
print(prediction[0])
```

Only load model artifacts that were generated by this project or another trusted source. `joblib` uses pickle-based serialization, so loading an untrusted file can execute unsafe code.

只加载由本项目或可信来源生成的模型文件。`joblib` 基于 pickle 序列化，加载不可信文件可能执行不安全代码。

The output is the predicted `max_supported_load_mbps`.

输出结果就是预测的 `max_supported_load_mbps`。

## Application Scenario / 应用场景

The model can support the optimization/orchestration phase:

该模型可以服务于后续优化与编排阶段：

```text
1. Orchestrator collects current traffic, resource allocation, and KPI metrics.
2. ML model predicts max_supported_load_mbps.
3. Optimizer compares current traffic with predicted capacity.
4. If traffic exceeds predicted capacity, the system can scale CPU/RAM/bandwidth or reschedule services.
```

```text
1. 编排器收集当前流量、资源配置和 KPI 指标。
2. ML 模型预测 max_supported_load_mbps。
3. 优化模块比较当前流量和预测容量。
4. 如果当前流量超过预测容量，系统可以扩容 CPU/RAM/带宽，或重新调度服务。
```

## Deploy on Zeabur / Zeabur 部署

This Streamlit app can be deployed to Zeabur with PostgreSQL and a persistent volume.

该 Streamlit 应用可以部署到 Zeabur，并接入 PostgreSQL 与持久化存储卷。

1. Create a Zeabur project and link this repository.
2. Add a PostgreSQL service.
3. Set variables in the app service:

1. 在 Zeabur 创建项目并连接本仓库。
2. 添加 PostgreSQL 服务。
3. 在应用服务中设置环境变量：

```text
DATABASE_URL=postgresql+psycopg://...
ARTIFACT_ROOT=/data/training_runs
```

4. Mount a persistent volume at `/data` so uploaded CSV files, models, and metrics survive restarts.
5. Deploy the latest `main` branch.

4. 将持久化 Volume 挂载到 `/data`，确保上传的 CSV、模型和指标文件在重启后仍然保留。
5. 部署最新的 `main` 分支。

The `zbpack.json` start command sets `PYTHONPATH=src` because this project uses a `src/` package layout and runs Streamlit directly from `src/ml_project/streamlit_app.py`.

`zbpack.json` 的启动命令设置了 `PYTHONPATH=src`，因为项目使用 `src/` 包结构，并直接从 `src/ml_project/streamlit_app.py` 启动 Streamlit。
